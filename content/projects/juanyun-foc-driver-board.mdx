## Project Brief / 项目简述

FOC 这条线一眼看上去很“硬”：STM32F4、DRV8301、PMSM、无感控制、Clarke / Park、SVPWM、电流采样、速度环和保护逻辑都在材料里出现了。也正因为这样，写它时要更克制。关键词可以吸引人，真正能站住的是板子怎么采样、怎么调制、怎么保护、怎么起转，以及这些过程有没有留下测试记录。

The folder contains a sensorless PMSM FOC board, STM32F4 firmware, DRV8301 driver work, motor-control source modules, datasheets, BOM/PnP exports, a STEP model, an EasyEDA project, and generated build folders. The page now treats those files as a material trail. It keeps the board readable while leaving room for the missing bring-up story.

源目录里有无感 PMSM FOC 板、STM32F4 固件、DRV8301 驱动相关代码、`motor/` 下的控制模块、数据手册、BOM/PnP、STEP、EasyEDA 工程和一堆生成目录。当前页面把这些文件当成材料线索来写：先让板子可读，再明确后面还要补上电、起转、波形和调参过程。

## Archive Scope / 归档范围

The useful lines in the folder are:

- Hardware: sensorless PMSM FOC driver board files, including selected BOM, pick-and-place, Gerber, STEP, and EasyEDA project outputs.
- Firmware: STM32F4 code around `adc`, `drv8301`, `foc_algorithm`, `hall_sensor`, `speed_pid`, and parameter-identification wrappers.
- Toolchain: IAR / Keil project material, CMSIS / STM32F4 library files, Debug / Release outputs, and vendor documentation in the raw folder.
- References: DRV8301, STM32F446RE, MOSFET, encoder, and CMSIS materials.

源目录里比较有用的线索有四类：

- 硬件：无感 PMSM FOC 驱动板，包含筛选后的 BOM、贴片坐标、Gerber、STEP 和 EasyEDA 工程输出。
- 固件：STM32F4 代码里和 `adc`、`drv8301`、`foc_algorithm`、`hall_sensor`、`speed_pid`、参数辨识相关的模块。
- 工具链：原始目录里的 IAR / Keil 工程、CMSIS / STM32F4 库、Debug / Release 输出和供应商文档。
- 参考资料：DRV8301、STM32F446RE、MOSFET、编码器和 CMSIS 相关材料。

## How The Files Are Used / 文件怎么用

The public file set answers "what existed" more clearly than "how well it worked." That is still useful. The schematic/manufacturing outputs, EasyEDA project, STEP model, driver references, and selected motor-control source files let the board be inspected from several angles.

But the next layer has to be runtime evidence: redrawn signal path, test screenshots, oscilloscope captures, current-sampling notes, startup-state explanation, fault handling, and tuning logs. Once those are filled in, this page can move from a board-and-source record toward a real FOC case study.

现在这组文件回答“当时有什么”比回答“跑得怎么样”更清楚。这样也有价值。原理图和制造输出、EasyEDA 工程、STEP 模型、驱动参考、选出来的电机控制源码，至少让这块板可以从几个角度被检查。

下一层要补的是运行证据：重画的信号链、测试截图、示波器波形、电流采样说明、起转状态、故障处理和调参日志。等这些补上以后，这页才会从板卡和源码记录，往真正的 FOC 案例靠近。

## What Stayed Out / 没放上来的东西

The raw folder also contains material that would make the page heavier without explaining the board better:

- full STM32F4 firmware source tree;
- IAR / Keil project folders;
- CMSIS / STM32F4 vendor library folders;
- Debug / Release build outputs;
- duplicate generated exports and tool caches.

原始目录里还有一些会让页面变重、但不太增加理解的内容：

- 完整 STM32F4 固件源码树；
- IAR / Keil 工程目录；
- CMSIS / STM32F4 vendor library；
- Debug / Release 构建输出；
- 重复生成文件和工具缓存。

## Missing Debug Notes / 还缺的调试记录

- A redrawn FOC signal path from phase current sampling to Park / Clarke transform, PI loops, SVPWM, and gate-driver output.
- Startup and protection notes: pre-charge, alignment, fault pin handling, over-current assumptions, and DRV8301 status reading.
- Test evidence: phase-current waveforms, PWM timing, speed response, failed startup cases, and parameter-tuning notes.

## Looking Back / 现在回头看

The main risk of this line is writing around vocabulary. FOC sounds impressive, but the useful writing lives in the plain checks: current sampling, modulation path, protection behavior, startup sequence, and test evidence. For now this page keeps the board files and the unanswered questions together, so the project does not pretend to be more mature than its records.

这条线最大的风险是围着名词写。FOC 听起来很厉害，但真正有用的记录还是那些朴素检查：电流采样、调制链路、保护行为、起转顺序和测试证据。现在这页先把板卡文件和还没回答的问题放在一起，别让项目看起来比记录本身更成熟。
