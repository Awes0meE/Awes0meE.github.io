## Project Brief / 项目简述

This page groups two thinner but related development lines: a solenoid-valve control archive and a single-phase BLDC quiet-fan driver archive. They are not large enough yet to become separate polished portfolio projects, but they are worth keeping because they show the actuator side of the hardware work: driving loads, reading simple inputs, showing state, and keeping a board archive from disappearing into a random folder.

源目录里能看到 STM32F103 valve-control code, OLED module references, EC11 encoder references, ULN2003 / stepper-driver material, a solenoid-valve driver EDA file, and a quiet-fan BLDC board BOM / EDA file。根据新的边界，这些 legacy 材料不是 Current_Product 敏感内容，所以筛选后的源码、原理图、BOM 和 EDA 可以作为公开证据；供应商示例包、可执行工具、完整工程树和构建输出仍然不直接发布。

## Archive Scope / 归档范围

- Solenoid-valve line: selected STM32F103 source files, IOC configuration, electromagnetic-valve control board context, GPIO / display / encoder / button style interaction.
- Display and input line: OLED and EC11 encoder module schematics, with vendor examples excluded.
- Driver reference line: ULN2003 Darlington array material, useful for thinking about low-side load drive and simple actuator wiring.
- Quiet-fan line: a single-phase BLDC fan-driver board archive with selected BOM and EasyEDA project evidence.

## Public Boundary / 公开边界

This archive still stays modest. The public files prove that the actuator and fan-driver lines existed, but they do not turn the archive into a complete release. Vendor source packages, executable tools, generated output, duplicate reference bundles, and uncurated project folders stay out of the website.

## What Would Make It Stronger / 后续补什么才像样

- A small state diagram for valve open / close / fault / manual input.
- A redrawn wiring diagram for actuator load, driver IC, MCU pins, and protection assumptions.
- Photos or screenshots that show OLED / encoder interaction without exposing private project files.
- For the BLDC fan board: target voltage, noise goal, topology, basic bring-up result, and test notes.

## Reflection / 复盘

这些边角材料不该删，也不该硬吹。执行器、风扇、OLED、编码器这些东西和嵌入式硬件方向是连着的，只是测试叙事还不够完整。现在先把能公开的 legacy 证据补上；等有测试记录和图，再决定要不要拆成正式 case study。
