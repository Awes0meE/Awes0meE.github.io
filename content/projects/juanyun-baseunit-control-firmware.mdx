## Project Brief / 项目简介

The base unit firmware is a smaller but cleaner embedded-control story than the AC unit hardware page. It focuses on how I organized a real STM32F103 firmware project so a desk cooling base can sense environment/outlet temperature, detect whether the device is present, drive a fan, display local state, report over Bluetooth, and remember the user's fan setting after power loss.

底座板固件这条线更偏软件工程。我把它整理成一个能独立讲清楚的 STM32 项目：传感、执行、显示、通信和参数持久化都在一个循环里协同工作，而不是单独的课堂 demo。

## My Work / 我的工作

- 我把主流程收敛到 `APP_BindHandles()`、`APP_Init()` 和 `APP_Loop()`，让 CubeMX 生成层和业务逻辑分开。
- 我整理了 `DHT11 / DHT22` 双路温湿度采集、红外在位检测、风扇 PWM、编码器/按键输入、OLED 显示和蓝牙上报。
- 我把 UI 状态拆成 `UI_OFF`、`UI_BOOT`、`UI_NORMAL`、`UI_CONFIRM`，避免关机确认、开机 Logo 和正常控制逻辑互相污染。
- 我加入了风扇设定值掉电记忆，使用内部 Flash 参数区、CRC32 和 commit 标记思路恢复最后一次有效值。
- 我把蓝牙上报改成 DMA 异步发送，降低周期上报阻塞主循环的风险。

## What This Shows / 这个项目能证明什么

This project is useful in the portfolio because it shows embedded product thinking at a small scale: timing, state, persistence, user interaction, and communication all need to behave together. The most important part is not that each driver exists, but that the system has clear ownership boundaries and can be explained by reading the code.
