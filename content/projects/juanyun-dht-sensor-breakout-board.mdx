## Project Brief / 项目简述

This is a small board task, but small does not mean empty. The plan defines a compact DHT11 / AM2302 breakout that can plug into a main control board with a simple 3-pin single-bus interface. The interesting part is not technical complexity. The useful part is how a tiny requirement becomes board size, connector, pin order, pull-up choice, decoupling, verification timing, and firmware constraints.

![DHT11 / AM2302 breakout plan cover](/uploads/projects/juanyun-public/dht11-am2302-breakout-plan-cover.png)

## Requirements Captured / 已明确的要求

- Board size target: `30 mm x 14 mm`.
- Connector: `PZ2.54` angled 3-pin header.
- Interface: `3V3 / DATA / GND`, single-wire communication.
- On-board support: pull-up resistor, power decoupling capacitor, and optional signal damping / filtering.
- Package target: maximum `0603` for passives.
- Mechanical anchor: one `M2` positioning hole.
- PCB: 2-layer board is enough.
- Supply preference: `3.3 V`.
- Verification: optional breadboard pre-check before PCB release, then soldered-board MCU validation.

## Sensor-Side Notes / 传感器侧注意点

The plan keeps DHT11 and AM2302 / DHT22 in the same task because both are humidity and temperature sensors with a single-wire-style data interface, but the package and pin mapping cannot be treated casually. AM2302 is wider than the board target, so the document allows the sensor body to slightly exceed the PCB edge while keeping the board outline fixed.

Firmware verification also matters: wait at least 2 seconds after power-on, avoid sampling faster than a 2-second interval, and check that humidity / temperature change in a reasonable direction when the sensor is touched or exposed to breath. It sounds basic, but this is exactly where small sensor boards often waste time.
