## Project Brief / 项目简述

This prototype has the texture of an idea being pushed into the real world before the folder structure caught up. There are 3D-printed air-channel parts, acrylic and cardboard routes, fan-selection experiments, ESP32 PWM control, a desktop monitoring program, PCB output, slicing files, a demo video, and a parts list all living close together.

这个原型很有早期项目的味道：风道结构还在试，控制链路也在试，桌面端显示也被塞进来。源目录里能看到 3D 打印版本、亚克力切割版本、纸箱低成本版本、ESP32 程序、桌面端程序、PCB 文件、演示视频和零件清单。它还没有到很规整的阶段，但已经把“风怎么走、风扇怎么控、数据怎么给人看”这几个问题拉到同一张桌子上。

## What Was In The Archive / 归档里有什么

The source folder reads like several routes were active at the same time:

- Structure route: Bambu Studio slicing previews, A1 mini / A1 / P1 / X1 print targets, split top and bottom shell parts, fan bracket, DC or Type-C interface covers, and support-setting screenshots.
- Low-cost route: acrylic-cutting and cardboard versions, useful for seeing how the airflow idea was tried before a cleaner enclosure existed.
- Control route: ESP32 PWM fan control, OLED-related dependencies, and a Windows desktop monitoring path.
- PCB route: EasyEDA projects, schematic export, Gerber package, acrylic parts list, and a Bambu Studio 3MF slicing file.
- Demo route: a Windows video shows the desktop side running; the installer and generated build output stay out of the site because they add noise rather than understanding.

源目录读起来像几条路线同时在跑：

- 结构路线：Bambu Studio 切片预览、A1 mini / A1 / P1 / X1 打印目标、上下壳拆件、风扇支架、DC 或 Type-C 接口盖，以及支撑设置截图。
- 低成本路线：亚克力切割和纸箱版本，用来观察风道想法在更粗糙条件下怎么被试出来。
- 控制路线：ESP32 PWM 风扇控制、OLED 相关依赖，以及 Windows 桌面监控程序。
- PCB 路线：EasyEDA 工程、原理图导出、Gerber、亚克力零件清单和 Bambu Studio 3MF 切片文件。
- 演示路线：桌面端运行视频可以说明链路曾经跑起来；安装包和生成的 build 输出不放进站点，它们增加的是噪声，不增加理解。

## What Stayed Out / 没放上来的东西

The useful material here is the part that explains the prototype chain. PlatformIO / Arduino snippets, C# entry points, PCB files, Gerber, EasyEDA, and 3MF are small enough to inspect and tied to the work. Installers, dependency folders, generated output, duplicate exports, vendor folders, and the uncurated full source tree would only make the page harder to read.

这条线真正有用的是能解释原型链路的材料。PlatformIO / Arduino 片段、C# 入口、PCB 文件、Gerber、EasyEDA 和 3MF 都比较小，也和项目直接相关。安装器、依赖目录、生成输出、重复导出、vendor 文件和没有整理过的完整工程树会把页面变得更难读。

## Looking Back / 现在回头看

The valuable trace is the collision between structure, airflow, electronics control, desktop display, and written instructions. That is exactly where early hardware prototypes become messy. Reading the hardware SOP later makes more sense after seeing this folder: small prototypes still need versions, interfaces, and verification notes. Without those traces, reopening the project becomes file-name archaeology.

这条线最值得保留的，是结构、风道、电子控制、桌面端显示和说明文档互相撞在一起的过程。早期硬件原型就是容易在这里变乱。后面再看硬件 SOP，会更容易理解为什么小项目也要留版本、接口和验证记录。没有这些痕迹，几个月后再打开项目，就只能靠文件名考古。
