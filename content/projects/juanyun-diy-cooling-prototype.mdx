## Project Brief / 项目简述

This prototype is messy in a very early-project way: structure, airflow, small electronics, ESP32 control, desktop monitoring, part sourcing, and user instruction material all live in the same archive. It is not a polished product page. The useful part is that it shows a complete idea being pushed through several real interfaces instead of stopping at one PCB or one sketch.

这条线的公开版本不把原始工程包整包挂出来，但会把能说明开发过程的关键证据补上。源目录里能看到 3D 打印版本、亚克力切割版本、纸箱低成本版本、ESP32 程序、桌面端程序、PCB 文件、演示视频和零件清单。根据新的边界，这不是 Current_Product 敏感项目，所以筛选后的 Gerber、EasyEDA、PlatformIO、Arduino、C# 和 3MF 证据可以公开；安装包、依赖目录、构建输出和混乱的完整工程树仍不直接发布。

## What Was In The Archive / 归档里有什么

- 3D printing route: Bambu Studio slicing previews, A1 mini / A1 / P1 / X1 print targets, split top and bottom shell parts, fan bracket, DC or Type-C interface covers, and support-setting screenshots.
- Low-cost structure route: acrylic-cutting and cardboard versions, mostly useful for seeing how the airflow idea was tested before a cleaner enclosure existed.
- Control route: ESP32 PWM fan control, OLED-related library dependencies, and a small desktop monitoring path.
- PCB and fabrication route: EasyEDA projects, a schematic export, a Gerber package, acrylic parts list, and one Bambu Studio 3MF slicing file.
- Delivery route: a Windows demo video and a desktop installer existed in the source archive, but the installer and build output are not published.

## Public Boundary / 公开边界

The prototype is now treated as a public legacy archive, not a sensitive current-product folder. Selected PCB, firmware, desktop, and fabrication files are published because they are small enough to inspect and tied directly to the prototype. The boundary is still deliberate: installers, dependency folders, generated build output, vendor folders, duplicate exports, and the uncurated source tree stay out of the website.

## Reflection / 复盘

它的价值不在“做得多成熟”，而在暴露了结构、风道、电子控制、桌面端显示和文档交付之间的混乱连接。后面再看硬件 SOP，会更容易理解为什么小项目也要留版本、接口和验证记录。没有这些东西，项目一多，回头看就只剩一堆不知道为什么这样命名的文件夹。
