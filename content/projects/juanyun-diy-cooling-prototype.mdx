## Project Brief / 项目简介

This prototype came before the more formal Juanyun thermal-control hardware work. I treated it as a hands-on cooling experiment: design a pressure-flow structure, choose fans and power components, add ESP32-side control, and build a small desktop-side monitoring workflow.

这个项目不是最终产品型工程，而是早期快速探索：我一边处理结构和风道，一边把风扇、电源、控制程序、桌面端显示和使用说明串起来。现在把它单独拆成项目页，是因为它比普通笔记更能展示我从机械结构到嵌入式控制再到桌面工具的完整动手链路。

![DIY pressure-flow cooling 3D print preview](/uploads/projects/juanyun-tech/diy-cooling-3d-print-preview-1.jpg)

## What I Built / 我做了什么

- I kept multiple enclosure directions: 3D printed parts, acrylic/CAD cutting files, a low-cost cardboard variant, and fan replacement notes.
- I organized the ESP32 control sketch and desktop-monitoring program as evidence of the control and display path.
- I preserved the schematic, demo video, and print-preparation screenshots so this prototype can be reviewed as a concrete build rather than a vague idea.

- 我保留了多条结构路线：3D 打印、亚克力切割、纸箱低成本版本和风扇替代方案。
- 我整理了 ESP32 控制程序和桌面端监控线索，说明这个项目不只是外壳结构，也包含控制和状态显示。
- 我把原理图、演示视频和打印准备截图放进网站，先形成可被浏览的证据链。

## Reflection / 复盘

Looking back, the main value of this prototype is not that it was polished. Its value is that it exposed all the messy interfaces between structure, airflow, electronics, software, and documentation. That experience later made the Juanyun hardware SOP and board-level documentation feel much more necessary.

回头看，这个项目最有价值的地方不是它多成熟，而是它把结构、风道、电子、电源、软件和文档之间的混乱连接暴露出来了。也正因为做过这种快速原型，我后来才更明确地意识到：硬件项目必须有需求、接口、版本、证据和复盘。
