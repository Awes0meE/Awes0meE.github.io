## Starting Point / 前期想法

SLAM looks like a single impressive keyword at first: mapping, localization, loop closure, robots. In practice it is a system problem. Sensors, compute budget, scene constraints, error accumulation, and real-time behavior all affect the result.

这篇不只抄公式。公式当然重要，但更重要的是先知道不同方法在真实机器人系统里各自怕什么、需要什么输入、失败模式是什么。

## Question / 疑问：为什么不先死磕某一个算法？

A single algorithm makes more sense after the system map is clear. Visual SLAM, LiDAR SLAM, IMU fusion, and wheel odometry provide different assumptions and failure modes. Without that map, it is easy to understand one paper while still not knowing where it belongs in a robot.

可以先看某个算法，但更缺的是系统地图。先知道视觉、激光、IMU、轮速计各自能提供什么，再去看具体算法会更稳。

## Stage Goals / 阶段目标

- Map the relationship between localization, mapping, and loop closure.
- Compare input requirements for visual SLAM, LiDAR SLAM, and multi-sensor fusion.
- Record failure scenarios instead of only successful demos.
- Connect the reading route to robot-platform or graduation-thesis work later.

## Reflection / 复盘

This note is currently a reading route map. SLAM is not only algorithm learning; it is also system tradeoff training: sensor choice, compute allocation, scene limitation, and failure diagnosis.
