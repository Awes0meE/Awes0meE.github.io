## Early Idea / 一开始怎么想

Seamly2D stores pattern data in XML-style files. Before touching deeper behavior, it made sense to open a real `.sm2d` file and a related `.smis` measurement file, then see what the application is actually reading.

Seamly2D 的版型文件本质上是 XML 结构。继续往里面做功能之前，我先找了一个 `.sm2d` 样例和对应的 `.smis` 量体文件，想看清楚软件到底在读什么。

## What I Looked For / 当时在看什么

当时主要想分清三类内容：几何点和曲线在哪里，量体变量在哪里，界面状态或元数据又在哪里。如果这些层次都没看明白，直接说“我要解析格式”其实会很空。

The sample files were useful as reading material. They showed that the project format is structured enough to inspect, but a real parser would still need a clear purpose: validation, migration, visualization, or feature extraction.

## Looking Back / 回头看

这个方向最后没有做成大功能，反而是合理的。先读样例，确认格式长什么样，再决定要不要继续写解析器，比一上来就造一个没有用途的 parser 更现实。

对实习阶段来说，它更像是在补业务上下文：Seamly2D 不是普通画图软件，图形背后还连着尺寸、版型和 XML 数据。
