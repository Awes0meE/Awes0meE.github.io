## Starting Point / 前期想法

Kalman filtering is easy to recognize and hard to internalize. Prediction, update, covariance, and gain all look reasonable on paper. The practical question is simpler: when should the estimate trust the model, and when should it trust the measurement?

这篇笔记不急着堆推导。先从一个小例子开始：传感器有噪声，模型也不完全可靠，滤波器夹在中间做取舍。这个直觉建立以后，再回头看公式会舒服很多。

## Question / 疑问：为什么不直接用现成库？

Libraries are useful, but blind use only hides the confusion. At minimum, the parameters should explain what happens when measurement noise increases, when model prediction drifts, and why the gain changes.

当然可以用库，但如果完全不知道参数含义，用库也只是换了一层包装。至少要知道测量噪声变大以后会发生什么，模型预测不准时会发生什么，增益为什么会变。

## Stage Goals / 阶段目标

- Run prediction and update with a one-dimensional example.
- Plot raw measurement, reference value, and estimated value together.
- Adjust noise parameters and observe whether the filter trusts model or measurement more.
- Add outlier and latency handling when real sensor data is introduced.

## Reflection / 复盘

The useful part is not memorizing formulas. The useful part is estimation thinking: real data is noisy, models are imperfect, and engineering decisions often live between the two.
