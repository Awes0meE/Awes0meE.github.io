## Why This Note Exists / 为什么写这篇

The base unit firmware does not live alone. It depends on real component limits: encoder behavior, fan-driving MOSFET selection, power input connector, 3.3 V conversion, clock source, and small switching components. I kept these datasheets as supporting evidence so the firmware page can point back to board-level constraints.

底座固件不是凭空存在的。编码器手感、风扇 PWM 输出、电源输入、3.3 V 降压、晶振、开关和三极管都会影响固件可用性和硬件验证方式。我把这些规格书整理成一篇支撑笔记，方便后续继续补 PCB 和测试记录。

## What I Mapped / 我关注的内容

- XL1509-3.3 关系到 3.3 V 电源路径和负载余量。
- EC11 编码器关系到旋钮调速、去抖、方向判断和 UI 状态机。
- AO4407 MOSFET、SS8050 BJT 和风扇 PWM 输出关系到执行端驱动边界。
- DC-005 电源座和滑动开关关系到用户操作、上电/断电路径和结构安装。
- 8 MHz 与 32.768 kHz 晶振资料用于确认时钟源和后续低功耗/RTC 预留讨论。

## Reflection / 复盘

This note is not a claim that I designed every component from scratch. It records how I treat datasheets as engineering inputs instead of random attachments. Later, when I add real PCB screenshots or measurement logs, this page can explain why those parts mattered.

这篇不是为了堆附件，而是为了说明我会把器件资料当成设计输入。后续补 PCB 或实测记录时，这些规格书能解释为什么某些接口、供电和固件处理方式要这样做。
