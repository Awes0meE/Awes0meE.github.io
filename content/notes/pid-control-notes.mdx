## Starting Point / 前期想法

PID looks simple in class: three letters, one formula, a few tuning rules. Real tuning feels different. A larger P term can become aggressive, integral action slowly pays back accumulated error, and D looks clever until noise starts fighting back.

这篇笔记先把 PID 从“理论知道一点”拉到“实验能说清楚”。只背公式没用，误差、输出、响应曲线和调参过程必须放在一起看。

## Question / 疑问：为什么 PID 需要可视化？

Only looking at the final result is misleading. Overshoot, oscillation, steady-state error, actuator saturation, and sluggish recovery all show up more clearly in curves than in a single success/failure result.

只看最终结果很容易误判。系统是不是超调、是不是振荡、是不是稳态误差太大、是不是执行器已经饱和，都要看曲线。没有可视化，调参就像闭眼拧螺丝，拧对了也不知道为什么。

## Stage Goals / 阶段目标

- Explain how P, I, and D separately affect response behavior.
- Record tuning as a process instead of only keeping the final parameters.
- Plot error, control output, and measured value together.
- Connect the note to PID Starter Kit as embedded experiment evidence.

## Reflection / 复盘

PID is not necessarily advanced, but it is very engineering-oriented. A useful PID note should eventually connect theory, code, waveform, hardware experiment, and tuning record into one reusable manual.
