## Why I Kept The Older Versions / 为什么保留早期版本

The V2.1 page shows the current direction, but the older exports are useful because they show iteration history. A portfolio case study is stronger when I can explain how a board moved from an earlier layout and mechanical export into a three-board architecture with clearer manufacturing files.

V2.1 是当前更完整的方向，但 V1.0 / V2.0 的资料能证明版本迭代过程。作品集不应该只放最终截图，也应该留下早期打样、结构、BOM、Gerber 和三板拆分的证据。

## Reflection / 复盘

I am keeping this as an archive note rather than over-polishing it now. The useful next step is to add a version comparison table explaining what changed between V1.0, V2.0, and V2.1: board split, connector choices, pressure sensor count, Wi-Fi reserve, power routing, and interaction-board layout.

这篇先作为版本证据归档。后续最值得补的是 V1.0 / V2.0 / V2.1 对比表：为什么拆板、为什么改接口、压力变送器数量怎么变、Wi-Fi 怎么预留、电源路径和交互板布局怎么变化。
